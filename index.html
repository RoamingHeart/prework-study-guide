HTML

  • the head element contains info about the webpage
  • the body element is what is shown on the webpage
  • head element is the container for metadata
  • meta element contains info about the webpage used by the browser
  • users do not see the head or meta elements

CSS

  • div is the selector, the things inside the curly brackets{} is the declaration, which is the rules for the code/ color and text-align are property, and blue and justify are value
  • example: div {color: blue; text-align: justify}
  • outside to inside: margin/border/padding/content
  • margin is the space outside the element, setting it to 0 means there is no extra space between elements
  • padding is space around the content, it also affects the size of the element, more padding = more element
  • can have more than one selector in a single CSS rule by seperating them with a comma, e.g. header,footer
  • need to link to html if you make a seperate css style sheet. e.g.: link rel="stylesheet" href="./assets/style.css"
  • class selectors needs a period "." (.card)(.hole) etc for css styling
  • box-shadow adds a shadow effect around the element's frame with pixel height, width, and color being changable. eg: xpx ypx color

Git

  • git status: checks what branch you are at
  • git checkout: changes what branch you are on
  • git checkout -b "insert-branch-name": creates and switchs to new branch
  • unset GIT_WORK_TREE: allows for cloning of repo when you get error fatal: working tree '...' already exists
  • git config --global core.autocrlf false: for git commit when warning: LF will be replaced by CRLF
  • git add -A: add new files to local git repo
  • git commit -m "insert comment here": commit changes with comments
  • git push: push changes to remote repo
  • git pull: fetch and download content from remote repo, and updates local repo to match

JavaScript

  • declare a varible using var, give the var a unique name, then assign a value to it
  • eg: var shape = "circle";
  • strings needs quotation marks or backticks to surround the data item. ''/""/``
  • strings only display what is in them
  • Boolean values, like numbers, do not need quotation marks, they just are
  • Boolean values only have two, tru and false
  • '===' operator means the two values must be equal to each other. eg: 'cat' === 'cat' is true
  • '++' is used in place of '+1'
  • arrays can hold multiple data items and can be of different types of data
  • eg: var fireMoth = ['Su', 50000, false]
  • previous example of fireMoth array is name is Su, age is 50000, and living is false (dead)
  • arrays need to be in square brackets []
  • array index is the order of the items inside the array, starting from 0
  • for loops adds conditions to help go through and array to get the result you need
  • eg of for loop: for(x = 0; x < fireMoth.length; x++){code block};
